Development

Most of these projects have been done with the Canvas API. JavaScript was also used to create user interactivity. Some projects focus on user experience and some projects are simple games made in Canvas.

To Cook or Not to Cook

To Cook or Not to Cook uses the Yummly API and Google Places API. This website allows for hungry and indecisive users to find a dish to make/cook for themselves or to find restaurants near them instead. The user has several options to play with in order to get several specific dishes to choose from.

Project 3 for IGME330, Fall 2016

I Like to Eat Too!!

I Like to Eat Too is a side-scrolling digital game created with the Canvas API. The user is a caterpillar whose goal is to eat leaves all while trying not to get eaten by the birds. All art and animations were created from scratch. The project heavily focused on the mastery of JavaScript.

Project 2 for IGME330, Fall 2016

Magical Visualizer

Magical Visualizer uses both the Canvas API and Web Audio API. The project is an audio visualizer made with HTML/CSS and JavaScript. Visuals are presented to the user based on the waveform and frequency of the music file. The user has several options in the controller to change how the music is represented on the canvas.

Project 1 for IGME330, Fall 2016

Other Projects

Having taken Game Design and Development as my minor, these projects show many design skills and coding concepts that I have developed. Some games where done alone and some where done in teams.

Asteroids

Asteroids was created in Processing 3. The project was to demostrate an understanding of vectors, vector-based movement and collission dection. Circle collision detection, acceleration, and deceleration was coded in order to give the ship some realistic movement in game. Goal of game is to shoot asteroids to gain points without colliding into them.

Project 2 for IGME202, Spring 2016

To be Determined by Combat

To be Determined by Combat is a group project made in Microsoft Visual Studio. Game mechanics invovles inputing battle commands during planning phase and watching the commands execute on screen after a certain amount of time has passed. All art was created from scratch.

Project 2 for IGME106, Fall 2015

Sushi Escape

Sushi Escape is a 2D Shmup game that was made in Unity. All art and animation was created from scratch. The goal for this side-scrolling shooting game was to create an opening scene with a creative back story.
